# Contributing

Thanks for helping improve Fusheng Paoying.

## Development Setup

```bash
npm install
npm run dev
```

Use Node.js 20 or newer. Source code lives in `src/`, tests live in `tests/`, and static assets live in `public/`.

## Checks

Run these before opening a pull request:

```bash
npm test
npm run build
```

For visual changes, also run the app locally and include screenshots or a short screen recording in the pull request.

## Pull Requests

- Keep each pull request focused on one behavior, fix, or asset update.
- Describe the user-visible change and the checks you ran.
- Do not commit personal media, local scratch files, `.env` files, or generated build output.
- Add asset credits near the asset when adding third-party media.
